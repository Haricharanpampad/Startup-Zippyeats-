💳 Webhook Simulator

Use this developer module to mock a successful payment gateway checkout event.

🎓 Developer Guide: What do these blanks do?

🔑 1. Active Order UUID This is the unique database identifier for the traveler's pending food preorder. When a user submits an order on the map, it starts in a PENDING state. The UUID ensures the webhook triggers the payment update on the exact correct order in the database.
💸 2. Mock Payment Intent ID This represents the unique transaction receipt ID from a payment processor (like Stripe or PayPal). It serves as proof of authorization, securing the traveler's money in Escrow. This receipt ID is linked to the order for auditing and refund purposes.
⚡ How this simulates a Real Webhook In production, when a customer pays, the payment gateway automatically broadcasts a server-to-server HTTP POST request (a "webhook") to this backend route. This tool simulates that callback, confirming the funds are secured in escrow, which instantly alerts the restaurant's kitchen to start preparing the fresh meal!
